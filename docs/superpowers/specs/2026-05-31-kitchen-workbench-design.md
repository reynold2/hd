# 制作端工作台设计

## 目标
将 `customer-miniapp` 的制作端页面从示例页改造成一个可用于真实制作流转的工作台，支持待制作、制作中、已完成三种状态，并按现有移动端视觉风格呈现。

## 范围
- 重做 `customer-miniapp/src/pages/kitchen/index.vue`
- 复用现有 API 封装层，必要时补充制作端所需接口
- 保持页面为移动端单页体验
- 先实现单门店默认入口，后续再考虑多门店或权限分流

## 页面结构
1. 顶部信息区
   - 页面标题：制作端 / 等锅叫号 · 制作首页
   - 简短说明：支持点选切换制作状态
2. 状态提示区
   - 提示用户：第一次点击进入制作中，第二次点击完成并触发叫号/流转
3. 订单卡片网格
   - 两列布局
   - 每张卡片显示订单号与标签信息
   - 待制作：白底橙字
   - 制作中：绿色底色
   - 完成：从首页移除，进入已完成列表
4. 底部导航
   - 制作首页
   - 已完成列表

## 数据与状态
- 页面初始化时从后端加载制作队列
- 前端维护三类状态：`pending`、`preparing`、`done`
- 点击卡片时更新本地状态，并调用对应后端接口同步
- 完成状态的数据不在首页列表中继续展示，只保留在已完成视图

## 接口约定
优先复用 `customer-miniapp/src/api.mjs` 中已有的请求封装。
若后端缺少接口，则补充最小可用接口，至少满足：
- 获取制作队列列表
- 更新订单状态
- 触发完成后的叫号/流转动作

## 交互细则
- 点击待制作卡片：切换为制作中
- 再次点击制作中卡片：标记完成并自动执行后续动作
- 交互完成后给出明确视觉反馈，避免用户误触
- 网络错误时保留当前状态并提示重试

## 验收标准
- 页面不再展示示例文案
- 订单卡片可根据状态切换颜色
- 第二次点击能够完成流转
- 底部导航可在首页与已完成视图之间切换
- 代码风格与现有 `customer-miniapp` 保持一致
